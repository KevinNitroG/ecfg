- `TokenEquals` — `=`
- `TokenValue` — value after `=`

- `Range Range`

  <action>
Create test fixture directories and files for parser validation:

**valid/** — Correct EditorConfig syntax:

1. `simple.editorconfig`: Single section with 2-3 properties
   ```
   [*.go]
   indent_style = tab
   indent_size = 4
   ```

2. `preamble.editorconfig`: Preamble with `root = true` + section
   ```
   root = true
   
   [*.js]
   indent_style = space
   ```

3. `multi-section.editorconfig`: Multiple sections with different globs
   ```
   [*.{js,ts}]
   indent_style = space
   
   [*.go]
   indent_style = tab
   ```

4. `complex-globs.editorconfig`: Advanced glob patterns
   ```
   [*.{js,jsx,ts,tsx}]
   [**/test/**/*.go]
   [lib/{src,test}/**/*.py]
   ```

5. `comments.editorconfig`: Both `#` and `;` comments, inline and standalone

**malformed/** — Invalid syntax for error recovery:

1. `unclosed-section.editorconfig`: `[*.go` (no closing `]`)
2. `missing-equals.editorconfig`: `indent_style tab` (no `=`)
3. `empty-key.editorconfig`: `= value` (no key before `=`)
4. `invalid-utf8.editorconfig`: Invalid UTF-8 byte sequences

**positions/** — Position tracking edge cases:

1. `multiline.editorconfig`: 50+ lines with varying line lengths
2. `unicode.editorconfig`: Multi-byte UTF-8 characters (emoji, CJK)
3. `mixed-whitespace.editorconfig`: Tabs, spaces, trailing whitespace

Ensure each file is realistic and tests a specific parser behavior per the research findings.
