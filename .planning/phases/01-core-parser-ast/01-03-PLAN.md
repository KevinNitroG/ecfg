Then implement `internal/parser/parser.go`:

**Parser struct:**
- `lexer *Lexer` — token source
- `current Token` — lookahead token
- `errors []ParseError` — collected parse errors

**Parse(source []byte) (*Document, error)** — main entry point
- Creates Lexer, calls parseDocument()
- Returns Document even if parse errors occurred (for LSP)
- Never panics

**parseDocument() *Document:**
1. Initialize empty Document
2. Parse leading comments (before any section) → add to Document.Comments
3. Parse preamble key-value pairs (before first section) → create Preamble
4. Parse sections in loop:
   - Detect TokenSectionStart → parseSection()
   - Add Section to Document.Sections
5. Return Document with all collected Errors

**parseSection() *Section:**
- Expect TokenSectionStart
- Read TokenIdentifier(s) until TokenSectionEnd (glob pattern)
- If no TokenSectionEnd before newline → add ParseError("missing-section-close"), recover
- Parse key-value pairs and comments until next section or EOF
- Return Section

**parseKeyValue() *KeyValue:**
- Expect TokenIdentifier (key)
- If next token not TokenEquals → add ParseError("missing-equals"), recover to next line
- Read TokenValue (may be empty if nothing after =)
- Trim whitespace from key/value per spec
- Store original Range for KeyRange and ValueRange
- Return KeyValue

**parseComment() *Comment:**
- Expect TokenComment
- Store full text (including # or ;)
- Return Comment with Range

**Error recovery strategy:**
- On unexpected token, add ParseError with current position
- Skip tokens until synchronization point (TokenNewline, TokenSectionStart, or TokenEOF)
- Continue parsing from sync point
- Never panic — always return partial AST + errors

**Synchronization points:**
- After ParseError for unclosed section → continue at next line
- After ParseError for missing equals → skip to next TokenNewline
- After ParseError for unexpected token → skip to next TokenNewline or TokenSectionStart

**Critical implementation details:**
- Trim leading/trailing whitespace from keys and values (per EditorConfig spec)
- Preserve internal whitespace in values
- Distinguish preamble (before first section) from section content (PARSE-02, PARSE-07)
- Store HeaderRange separately for section diagnostics (Phase 2)
- Track Document.Range as Range{Start: Position{0,1,0}, End: Position{len(source), lastLine, lastCol}}
