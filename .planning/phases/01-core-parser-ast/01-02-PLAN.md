Then implement `internal/parser/lexer.go`:

**Lexer struct:**
- `source []byte` — input text
- `pos int` — current byte offset
- `line int` — current line (1-indexed)
- `col int` — current column (0-indexed)

**NewLexer(source []byte) *Lexer** — constructor

**Scan() Token** — returns next token, advances position

**State machine logic:**
1. Skip leading whitespace on line (but track position)
2. Detect token start character:
   - `#` or `;` → read to end of line, emit TokenComment
   - `[` → emit TokenSectionStart
   - `]` → emit TokenSectionEnd
   - `=` → emit TokenEquals
   - `\n` or `\r\n` → emit TokenNewline, increment line, reset column
   - Otherwise → read identifier or value based on context

**Position tracking:**
- Track `offset`, `line`, `col` as you scan
- Each Token.Range = Range{Start: posAtTokenStart, End: currentPos}
- Increment `line` on `\n` or `\r\n`, reset `col` to 0
- Increment `col` for each rune (use `utf8.DecodeRune` to handle multi-byte)

**Error recovery:**
- If unclosed section (no `]` before newline) → emit what you have, continue
- If invalid UTF-8 → skip byte, emit error token or continue
- Never panic — always return valid Token (even if TokenEOF)

**Critical:** Use `utf8.DecodeRune` to correctly handle multi-byte UTF-8 (affects column counting).

Test with fixtures from Plan 01 (`testdata/valid/`, `testdata/malformed/`, `testdata/positions/`).
